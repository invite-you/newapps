================================================================================
앱 마켓 데이터 수집 시스템 - 데이터베이스 스키마 및 실제 데이터 예시
================================================================================

[개요]
이 시스템은 Google Play Store와 Apple App Store에서 앱 정보를 수집하여 SQLite 데이터베이스에 저장합니다.
- 메인 데이터베이스: apps.db (앱 상세 정보)
- 추적 데이터베이스: sitemap_tracking.db (수집 이력 관리)

================================================================================
1. apps.db - apps 테이블 (총 66개 컬럼)
================================================================================

[1.1] 기본 식별 정보
--------------------------------------------------------------------------------
컬럼: id (INTEGER, PRIMARY KEY)
설명: 내부 자동 증가 ID
예시:

컬럼: app_id (TEXT, NOT NULL)
설명: 앱 스토어 고유 식별자. Google Play는 패키지명, App Store는 숫자 ID

5. 데이터 정규화 규칙
================================================================================

날짜 정규화:
- 모든 날짜는 ISO 8601 형식으로 저장 (YYYY-MM-DD 또는 YYYY-MM-DDTHH:MM:SSZ)

가격 정규화:
- 숫자는 REAL 타입으로, 표시 문자열은 TEXT로 별도 저장

평점 정규화:
- 1.0 ~ 5.0 범위로 정규화
- NULL이 아닌 경우에만 저장

설치 수 정규화:
- 텍스트 형식 (예: "1,000,000+")과 숫자 형식 모두 저장

국가 코드 정규화:
- 2자리 소문자 ISO 국가 코드 (예: us, kr, jp, tw)
